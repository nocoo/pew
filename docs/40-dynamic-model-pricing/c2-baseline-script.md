# C2 — Root sync-prices script + bundled baseline JSON

## Scope

Add the developer-facing one-shot CLI that fetches OpenRouter + models.dev, runs them through the C1 pure layer, and writes a deterministic `model-prices.json` baseline that worker-read will bundle at deploy time. Plus the regression test that pins this baseline to the existing `DEFAULT_MODEL_PRICES` table so C5's cutover cannot lose coverage.

Still **no** runtime behavior change. Worker, web, and `lib/pricing.ts` remain untouched.

## Module contracts

### `scripts/sync-prices.ts`

CLI entry:

```bash
bun run sync-prices                        # fetch live, write baseline
bun run sync-prices --dry-run              # fetch + merge, print summary, do not write
bun run sync-prices --fixture <dir>        # read JSON from local dir instead of network (used by tests)
```

Exit codes:
- `0` success.
- `1` network/parse error severe enough that no entries were produced.
- `2` schema/regression failure (output would lose a model present in the prior baseline).

Internal shape:

```typescript
interface SyncOptions {
  dryRun: boolean;
  fixtureDir: string | null;   // when set, read openrouter.json + models-dev.json from this dir
  outputPath: string;          // defaults to packages/worker-read/src/data/model-prices.json
  now: string;                 // ISO-8601, injected for determinism in tests
}

interface SyncResult {
  entries: DynamicPricingEntry[];
  meta: DynamicPricingMeta;
  warnings: string[];
}

export async function runSync(opts: SyncOptions): Promise<SyncResult>;
```

`runSync` composition:

1. **Fetch** — when `fixtureDir` is null, GET `https://openrouter.ai/api/v1/models` and `https://models.dev/api.json` in parallel with a 30 s timeout each. Both must succeed; any failure returns exit 1.
2. **Parse** — `parseOpenRouter(json, now)` and `parseModelsDev(json, now)` from C1.
3. **Read prior baseline** — if `outputPath` already exists, load and pass as `baseline` to merge. (First-run case: `baseline = []`.)
4. **Merge** — `mergePricingSources({ baseline, openRouter, modelsDev, admin: [], now })`. C2 has no admin overlay.
5. **Regression check** (always, even in `--dry-run`) — assert every model ID present in the prior baseline JSON is still present in the new entries. If any model would be dropped, exit 2 with the list. (External APIs occasionally drop a model temporarily; we refuse to silently shrink the baseline.)
6. **Write** — only when `dryRun === false`. Serialize with `JSON.stringify(entries, null, 2) + '\n'`. Output is sorted by `[provider, model]` (already enforced by C1's merge), so re-running with no upstream changes produces a zero-diff file.

CLI argument parsing uses `Bun.argv` directly — no extra dep.

The script is self-contained: imports only from `packages/worker-read/src/sync/*` (the C1 modules) and standard library. It is **not** part of any worker bundle.

### `packages/worker-read/src/data/model-prices.json`

Raw shape: `DynamicPricingEntry[]` (the same array the C1 `merge` produces). No wrapper object; the bundled meta is materialized at read time inside the worker (C3) using the file's git mtime as a soft hint.

Initial content for this commit:

- Generated by running `bun run sync-prices --fixture scripts/__tests__/fixtures/` once locally against checked-in fixtures, **then hand-augmented** to ensure all 14 `DEFAULT_MODEL_PRICES` entries appear with identical `input` / `output` / `cached` values. The augmented entries get `origin: 'baseline'` and `updatedAt: <commit timestamp>`.
- Subsequent live runs (`bun run sync-prices`) replace those 14 entries with whatever upstream returns, but the regression test guarantees they remain present.

### Test fixtures

`scripts/__tests__/fixtures/openrouter.json` and `scripts/__tests__/fixtures/models-dev.json` — small (≤5 entries each), hand-crafted JSON that exercises the merge path. Distinct from C1's parser fixtures so this test doesn't accidentally pin parser behavior.

## Tests

### `scripts/__tests__/sync-prices.test.ts`

```typescript
test('fixture mode produces deterministic output')
test('dry-run does not write file')
test('regression check fails when prior baseline has model that fixture lacks')
test('first-run (no prior baseline) writes successfully')
test('warnings from parsers propagate into SyncResult')
```

Network is never exercised — every test passes `fixtureDir`. Live-fetch path is covered manually + by a separate (non-CI) smoke task.

### `packages/worker-read/src/data/model-prices.test.ts`

Pure data validation, no I/O beyond reading the JSON file:

```typescript
test('all 14 DEFAULT_MODEL_PRICES models are present with identical pricing')
test('every entry conforms to DynamicPricingEntry schema')
test('entries are sorted by [provider, model]')
test('no duplicate model IDs')
```

The first test imports `DEFAULT_MODEL_PRICES` from `packages/web/src/lib/pricing.ts` and walks every entry; for each, it finds the matching baseline entry (by exact `model` key) and asserts `inputPerMillion === input`, `outputPerMillion === output`, `cachedPerMillion === cached ?? null`. This is the regression guard that lets C5 swap the lookup source without surprising users.

## Conventions followed

- Script file is in `scripts/`, alongside existing `release.ts` and `clear-d1-usage.ts` — same import style and `Bun.argv` pattern.
- JSON is committed with trailing newline + 2-space indent, matching `tsconfig.json` style in the repo.
- All tests use `vitest` and live next to the file they cover.
- Regression test failure messages list the missing models so the dev knows exactly what upstream dropped.

## What this commit does NOT do

- Does not register `model-prices.json` as a worker import (C3 does that).
- Does not add a wrangler binding, KV write, or scheduled handler.
- Does not change `lib/pricing.ts`, the cost path, or any UI.
- Does not run live fetches in CI — fixture mode only.
- Does not invalidate any cache.
